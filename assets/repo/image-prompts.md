# Repository visual prompts

These prompts are for public repository presentation assets. They do not contain private account data, article drafts, or API credentials.

## `hero.jpg`

用途：README 顶部横幅，展示“微信公众号内容工作室”的整体气质。

正向提示词：

```text
生成一张 2.35:1 横版高质量中文科技媒体风格仓库封面图，用于 GitHub README 顶部展示。画面主题是“微信公众号内容工作室”：前景是一张干净的编辑台桌面，桌面上有笔记本电脑、手机预览窗口、文章卡片、数据图表卡片和图片生成画板；中景用半透明流程线把“选题、写作、视觉、排版、发布”五个环节连接起来；背景是柔和的深蓝到青绿色渐变工作室空间，有微弱网格和纸张纹理，但不能像 PPT 模板。整体视觉要像经过认真设计的中文商业媒体产品头图，现代、专业、有层次、留白充足。可以出现少量清晰简体中文标签：“选题”“写作”“视觉”“排版”“发布”，但不要密集文字。光线从左上方柔和照射，桌面有真实阴影和轻微反射，材质有玻璃、纸张、金属和屏幕光。画面必须有可见主体和空间纵深，不能只是纯色背景加大字，不能是廉价科技光效。
```

负向提示词：

```text
不要 PPT 模板感，不要纯渐变背景，不要只有巨大标题，不要空白背景，不要廉价霓虹光效，不要乱码中文，不要错别字，不要繁简混用，不要英文乱入，不要密集小字，不要低清晰度，不要脏背景，不要无意义图标，不要过度装饰。
```

## `workflow.jpg`

用途：README 工作流示意图，展示从热点到草稿箱的链路。

正向提示词：

```text
生成一张 16:9 横版高质量中文信息图，用于 GitHub README 展示产品工作流。画面主体是一条从左到右的清晰流程河流或流水线，分成六个节点：热点、素材、写作、视觉、排版、草稿箱。每个节点是一张有质感的卡片，卡片内有简洁图形隐喻：热点是热搜榜和放大镜，素材是资料夹和数据片，写作是编辑器文档，视觉是 image2 生成画板，排版是手机公众号预览，草稿箱是微信绿色收纳盒。整体为中文商业媒体信息图风格，白底偏暖，蓝绿色主色，少量橙色强调，卡片有轻微投影和纸张纹理，信息清楚但不拥挤。必须使用简体中文，节点文字清晰准确，仅出现“热点、素材、写作、视觉、排版、草稿箱”这些短标签。画面必须像完整设计稿，不像 PPT 截图，不要只有背景和大字。
```

负向提示词：

```text
不要 PPT 模板感，不要空白图，不要纯色背景加大字，不要乱码中文，不要错别字，不要繁简混用，不要英文乱入，不要错误流程顺序，不要密集小字，不要低清晰度，不要卡片堆叠过乱，不要无意义图标。
```
